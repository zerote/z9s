# z9s v0.55.6 — ArgoCD Integration

This release brings full **ArgoCD management** to z9s, mirroring the FluxCD experience: a visual overview, an Applications browser styled after the ArgoCD web UI, a hierarchical resource tree with live manifest inspection, and real GitOps actions (Sync / Refresh) — all without leaving the terminal.

## ✨ New Features

### ArgoCD Overview (`<a>` from the Operators page)
- **Stacked vertical bar charts** for Application **Health** (Healthy, Progressing, Degraded, Suspended, Missing, Unknown) and **Sync** (Synced, OutOfSync, Unknown) status.
- **ArgoCD Events panel** with the latest ArgoCD-related Kubernetes events, scrollable with the arrow keys.
- If ArgoCD is the only operator installed, `Ctrl+O` jumps straight to its Overview.

### Applications browser (`<a>` from the ArgoCD Overview)
- List styled after the **ArgoCD web UI**: neutral rows with inline colored status icons.
- Columns: **NAME, PROJECT, SOURCE, DESTINATION, REVISION, SYNC, HEALTH, AGE**.

### Application resource tree (`Enter` on an Application)
- Replaces the default describe view with a **hierarchical tree** of every resource managed by the Application — including children resolved live from the cluster via `ownerReferences` (e.g. Application → Deployment → ReplicaSet → Pod).
- **Color-coded nodes** based on live resource status (pod readiness, phases, health).
- `Space` expands/collapses nodes, `r` refreshes the tree.

### Resource inspection
- `Enter` on any tree node shows the **live YAML manifest** of that resource.
- `x` on a **Secret** node shows its data **base64-decoded**.

### GitOps actions on Applications
- **Sync** (`<s>`) — triggers a real synchronization by setting the Application's `operation` field, exactly what `argocd app sync` does under the hood. Includes a confirmation dialog, and the sync shows up in ArgoCD as initiated by `z9s`.
- **Refresh** (`<r>`) — forces ArgoCD to re-poll git via the `argocd.argoproj.io/refresh` annotation.
- Both actions respect read-only mode.

## 🐛 Bug Fixes

- **Operator detection is now context-aware**: the `FluxCD: on/off - ArgoCD: on/off` header line was detected once at startup and cached forever, showing stale values after switching contexts. Detection is now cached per context and re-runs whenever the active context changes.

## 🧪 Testing

- Added ArgoCD demo Application manifests (`examples/argocd/`) covering Healthy, Progressing, Degraded, Suspended, OutOfSync and Unknown states, including `ConfigMap` and `Secret` resources to exercise the inspection views.

## ⌨️ New Shortcuts

| Key | Context | Action |
|-----|---------|--------|
| `<a>` | Operators page | Open the ArgoCD section |
| `<o>` | ArgoCD | Overview (Health/Sync charts + ArgoCD events) |
| `<a>` | ArgoCD Overview | Applications list |
| `Enter` | Applications | Open the Application resource tree |
| `<s>` | Applications | Sync the selected Application |
| `<r>` | Applications | Refresh the selected Application |
| `Enter` | Resource tree | View the live YAML manifest |
| `x` | Resource tree | Decode a Secret (base64) |
| `Space` | Resource tree | Expand/collapse node |
